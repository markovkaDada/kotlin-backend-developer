### 1. Управление клиентами:

1.1. **Создание клиента:**

- **Источник требования:** Владелец производства (Админ).
- **Описание требования:** Реализовать функционал создания клиента с обязательными полями:
    - Имя клиента (обязательное поле)
    - Название компании (обязательное поле)
    - Электронная почта (обязательное поле)
    - Номер телефона
- **Приоритет требования:** Высокий.

1.2. **Редактирование и удаление клиента:**

- **Источник требования:** Владелец производства (Админ).
- **Описание требования:** Предоставить возможность редактирования контактных данных клиента и удаления клиента из системы.
- **Приоритет требования:** Высокий.

1.3. **Просмотр списка клиентов:**

- **Источник требования:** Владелец производства (Админ).
- **Описание требования:** Обеспечить отображение списка всех клиентов с возможностью поиска по имени и компании.
- **Приоритет требования:** Высокий.

### 2. Управление заказами:

2.1. **Создание заказа:**

- **Источник требования:** Заказчик (клиент компании).
- **Описание требования:** Реализовать функционал создания заказа на вырезку деталей. Поля заказа включают:
    - Описание детали/заказа (обязательное поле)
    - Сумма заказа (обязательное поле)
    - Файл для резки (обязательное поле, например detail.dxf)
- **Приоритет требования:** Высокий.

2.2. **Просмотр списка заказов:**

- **Источник требования:** Владелец производства (Админ), Оператор ЧПУ, Заказчик.
- **Описание требования:** Обеспечить отображение списка заказов с фильтрацией по статусу. Для каждого заказа отображаются: клиент, описание, сумма, статус, дата создания.
- **Приоритет требования:** Высокий.

2.3. **Скачивание файла для резки:**

- **Источник требования:** Оператор ЧПУ.
- **Описание требования:** Предоставить возможность скачивания прикреплённого файла для резки из карточки заказа.
- **Приоритет требования:** Высокий.

### 3. Жизненный цикл заказа:

3.1. **Подтверждение заказа:**

- **Источник требования:** Владелец производства (Админ).
- **Описание требования:** Реализовать перевод заказа из статуса NEW в CONFIRMED после проверки заказа администратором.
- **Приоритет требования:** Высокий.

3.2. **Взятие заказа в работу:**

- **Источник требования:** Оператор ЧПУ.
- **Описание требования:** Обеспечить перевод заказа из статуса CONFIRMED в IN_PROGRESS при начале выполнения оператором.
- **Приоритет требования:** Высокий.

3.3. **Завершение заказа:**

- **Источник требования:** Оператор ЧПУ.
- **Описание требования:** Реализовать перевод заказа из статуса IN_PROGRESS в COMPLETED после выполнения резки.
- **Приоритет требования:** Высокий.

3.4. **Отмена заказа:**

- **Источник требования:** Владелец производства (Админ).
- **Описание требования:** Предоставить возможность отмены заказа (перевод в статус CANCELLED) на любом этапе до завершения.
- **Приоритет требования:** Высокий.

### 4. Уведомления:

4.1. **Уведомления о смене статуса:**

- **Источник требования:** Заказчик (клиент компании).
- **Описание требования:** Обеспечить отправку уведомлений заказчику при каждой смене статуса его заказа через Kafka и Notification Service (email/push).
- **Приоритет требования:** Средний.

### 5. API Gateway:

5.1. **Маршрутизация и аутентификация:**

- **Источник требования:** Все типы пользователей.
- **Описание требования:** Обеспечить единую точку входа для всех запросов с маршрутизацией к бэкенд-сервисам, аутентификацией пользователей и ограничением частоты запросов (rate limiting).
- **Приоритет требования:** Высокий.
