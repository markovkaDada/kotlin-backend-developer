### 1. Производительность:

1.1. **Быстродействие API:**

- **Источник требования:** Оператор ЧПУ, Заказчик.
- **Описание требования:** Обеспечить высокую скорость обработки запросов к REST API, чтобы минимизировать ожидание пользователей.
- **Параметры:** Среднее время ответа API не более 500 мс для операций чтения, не более 1 секунды для операций записи.
- **Приоритет требования:** Высокий.

1.2. **Загрузка и скачивание файлов:**

- **Источник требования:** Заказчик, Оператор ЧПУ.
- **Описание требования:** Обеспечить стабильную загрузку и скачивание файлов для резки (DXF и др.) без таймаутов.
- **Параметры:** Поддержка файлов размером до 50 МБ.
- **Приоритет требования:** Высокий.

### 2. Надёжность:

2.1. **Стабильность работы:**

- **Источник требования:** Владелец производства (Админ).
- **Описание требования:** Гарантировать стабильную работу сервиса без сбоев и потери данных заказов.
- **Параметры:** Доступность сервиса не менее 99.5% времени.
- **Приоритет требования:** Высокий.

2.2. **Гарантия доставки событий:**

- **Источник требования:** Заказчик.
- **Описание требования:** Обеспечить гарантированную доставку событий смены статуса через Kafka в Notification Service без потери сообщений.
- **Параметры:** At-least-once семантика доставки сообщений.
- **Приоритет требования:** Высокий.

### 3. Безопасность:

3.1. **Аутентификация и авторизация:**

- **Источник требования:** Владелец производства (Админ).
- **Описание требования:** Обеспечить разграничение доступа по ролям (Админ, Оператор, Заказчик) через API Gateway. Каждая роль имеет доступ только к разрешённым операциям.
- **Параметры:** Аутентификация через JWT-токены, HTTPS для всех соединений.
- **Приоритет требования:** Высокий.

3.2. **Защита данных:**

- **Источник требования:** Заказчик.
- **Описание требования:** Обеспечить защиту персональных данных клиентов и конфиденциальность загруженных файлов для резки.
- **Параметры:** Хранение паролей в хешированном виде, передача данных по шифрованному каналу.
- **Приоритет требования:** Высокий.

### 4. Масштабируемость:

4.1. **Горизонтальное масштабирование:**

- **Источник требования:** Владелец производства (Админ).
- **Описание требования:** Обеспечить возможность горизонтального масштабирования бэкенда и Notification Service при росте числа заказов.
- **Параметры:** Stateless-архитектура бэкенда, возможность запуска нескольких экземпляров за API Gateway.
- **Приоритет требования:** Средний.

### 5. Сопровождаемость:

5.1. **Логирование и мониторинг:**

- **Источник требования:** Владелец производства (Админ).
- **Описание требования:** Обеспечить структурированное логирование всех операций и мониторинг состояния сервисов.
- **Параметры:** Логирование в JSON-формате, метрики доступности и времени ответа.
- **Приоритет требования:** Средний.
